# 团队协作开发指南

**项目名称**: CareLinkAgent - 智能医疗预诊系统\
**仓库地址**: https://github.com/Felicity-byte/-.git

***

## 一、分工

| 开发者 | 角色 | 负责模块 | 工作方式 |
| --- | --- | --- | --- |
| **开发者 A** | 主要开发者 | GlmAI + Backend | 直接push到main |
| **开发者 B** | 协作开发者 | Frontend | 分支开发 + PR |

***

## 二、开发者 A

### GitHub上要做的事

1. **邀请开发者B**
   - 打开 https://github.com/Felicity-byte/-/settings/access
   - 点击 **Add people**
   - 输入开发者B的GitHub用户名或邮箱
   - 选择 **Write** 权限，发送邀请

2. **审核开发者B的PR**
   - 收到PR通知后，打开仓库
   - 点击 **Pull requests** → 查看PR
   - 点击 **Files changed** 查看代码改动
   - 确认无误后点击 **Merge pull request**

### 日常开发

```bash
git pull origin main
# 修改代码...
git add .
git commit -m "feat: 描述"
git push origin main
```

***

## 三、开发者 B

### GitHub上要做的事

1. **接受邀请**
   - 查收邮件，点击邀请链接
   - 或访问 https://github.com/Felicity-byte/- 接受邀请

2. **创建Pull Request**
   - 推送代码后，打开GitHub仓库
   - 点击 **Compare & pull request**
   - 填写描述，点击 **Create pull request**
   - 等待开发者A审核

### 第一次使用

详见 [开发者B环境搭建指南.md](./开发者B环境搭建指南.md)

### 后续使用

```bash
# 1. 更新代码
git checkout main
git pull origin main

# 2. 创建分支
git checkout -b feature/功能名

# 3. 开发（只修改Frontend目录下的文件）
# ...

# 4. 提交
git add .
git commit -m "feat: 描述"
git push -u origin feature/功能名

# 5. 在GitHub创建Pull Request，等待开发者A审核
```

### PR被合并后

```bash
git checkout main
git pull origin main
git branch -d feature/功能名
```

***

## 四、注意事项

**开发者 B**: 不要直接push到main，每次开发前先pull
