# CareLinkAgent 后端框架文档

**项目名称**: MediMeow Backend API (智能医疗预诊系统)

**数据库**: MySQL + Tortoise-ORM
**认证方式**: JWT Token + Bearer

## 核心模块说明

### 1. 应用入口 (main.py)

| 功能 | 说明 |
|------|------|
| FastAPI实例创建 | 配置标题、描述、版本 |
| 中间件注册 | CORS跨域支持 |

| 异常处理 | HTTP异常、请求验证异常 |

### 2. 配置管理 (config.py)

```python
class Settings(BaseSettings):
    # 数据库配置
    DB_HOST: str
    DB_PORT: int
    DB_USER: str
    DB_PASSWORD: str
    DB_NAME: str

    # JWT配置
    SECRET_KEY: str
    ALGORITHM: str = "HS256"
    ACCESS_TOKEN_EXPIRE_MINUTES: int = 30

    # AI服务配置
    GLM_API_KEY: str
    AI_SERVICE_HOST: str = "127.0.0.1:50051"

    # CORS配置
    ALLOWED_ORIGINS: str = "http://localhost:3000,http://localhost:5173"
```

### 3. 数据库层 (app/models/)

#### 数据表关系图

```
Department (科室)
    ├── 1:N → Doctor (医生)
    │           └── 1:N → User (患者) [responsible_doctor]
    │
User (用户/患者)
    └── N:1 → Doctor (负责医生)
```

#### 模型字段说明

**Doctor (医生表)**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | CharField(36) | 主键UUID |
| username | CharField(100) | 医生姓名 |
| password | CharField(255) | 密码哈希 |
| department | ForeignKey | 所属科室 |
| phone_number | CharField(20) | 手机号（唯一） |
| title | CharField(50) | 职称 |
| status | CharField(20) | 状态(active/inactive) |

**User (患者表)**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | CharField(36) | 主键UUID |
| phone_number | CharField(20) | 手机号（唯一） |
| password | CharField(255) | 密码哈希 |
| responsible_doctor | ForeignKey | 负责医生 |
| gender/birth/ethnicity | - | 个人信息 |

**Department (科室表)**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | CharField(36) | 主键UUID |
| department_name | CharField(100) | 科室名称 |
| description | TextField | 科室描述 |

## API路由说明

### /user - 用户模块

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /user/register | 患者注册 | 否 |
| POST | /user/login | 患者登录 | 否 |
| POST | /user/bind | 绑定负责医生 | 是 |
| POST | /user/refresh | 刷新Token | 是 |

### /doctor - 医生模块

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /doctor/register | 医生注册 | 否 |
| POST | /doctor/login | 医生登录 | 否 |
| GET | /doctor/info | 获取医生信息 | 是 |
| POST | /doctor/patient/register | 注册新患者 | 是 |
| GET | /doctor/patients | 获取患者列表 | 是 |
| POST | /doctor/patient/bind | 绑定患者 | 是 |
| POST | /doctor/refresh | 刷新Token | 是 |

### /department - 科室模块

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | /department/list | 获取所有科室 | 否 |
| POST | /department/create | 创建科室 | 是 |

## 认证流程

### 登录流程

```
前端发送手机号+密码
    ↓
后端验证密码(bcrypt)
    ↓
生成 Access Token (30分钟) + Refresh Token (7天)
    ↓
返回双Token给前端
```

### Token刷新流程

```
前端携带Refresh Token
    ↓
后端验证Refresh Token有效性
    ↓
生成新的Access Token + Refresh Token
    ↓
返回新的双Token
```

### 依赖注入链

```
HTTPBearer (security)
    ↓
get_current_user() → 验证JWT → 返回{user_id, user_type}
    ↓
get_current_doctor() → 验证user_type == "doctor"
```

## 统一响应格式

### 成功响应

```json
{
    "base": {
        "code": "10000",
        "msg": "success"
    },
    "data": { ... }
}
```

### 错误响应

```json
{
    "base": {
        "code": "30001",
        "msg": "错误信息"
    }
}
```

### 错误码规范

| 代码范围 | 类型 |
|----------|------|
| 10000 | 成功 |
| 10001-10099 | 系统通用错误 |
| 30001-30099 | 医生模块错误 |
| 40001-40099 | 用户模块错误 |
| 50001-50099 | 科室模块错误 |

## AI服务集成

### 架构设计

```
Backend (FastAPI)
    ↓ gRPC
AI Service (Python/gRPC)
    ↓ HTTP
智谱GLM API
```

### gRPC通信

- **Proto文件**: medical_ai.proto
- **生成文件**: medical_ai_pb2.py, medical_ai_pb2_grpc.py
- **服务地址**: 127.0.0.1:50051
