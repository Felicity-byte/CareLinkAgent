## 快速启动

### Windows PowerShell

```powershell
# 1. 启动后端服务
cd Backend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 2. 启动前端服务 (新终端)
cd Frontend
npm run dev

# 3. 启动 AI 服务 (新终端)
cd GlmAI\connect
python server.py
```

| 前端 | 5173 | http://localhost:5173 |
| AI gRPC | 50053 | localhost:50053 |

## 详细启动步骤

### 1. 后端服务 (FastAPI)

#### 1.1 安装依赖

```bash
cd Backend
pip install -r requirements.txt
```

#### 1.2 配置环境变量

创建 `Backend/.env` 文件：

```env
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=carelink

# JWT 配置
SECRET_KEY=your_secret_key_here
ACCESS_TOKEN_EXPIRE_MINUTES=30

# 调试模式
DEBUG=True
```

#### 1.3 数据库迁移

```bash
# 方式一：使用 Aerich
aerich upgrade

# 方式二：通过 API 接口（仅开发环境）
# 访问 http://localhost:8000/migrate
```

#### 1.4 启动服务

```bash
# 开发模式（热重载）
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 生产模式
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

#### 1.5 验证启动成功

```bash
# 健康检查
curl http://localhost:8000/health

# 预期返回
{"status": "healthy", "service": "MediMeow Backend"}
```

---

### 2. 前端服务 (Vue.js)

#### 2.1 安装依赖

```bash
cd Frontend
npm install
```

#### 2.2 配置环境变量

创建 `Frontend/.env.local` 文件（可选）：

```env
VITE_API_BASE_URL=http://localhost:8000
```

#### 2.3 启动开发服务器

```bash
npm run dev
```

启动成功后，终端显示：

```
  VITE v4.5.14  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

#### 2.4 构建生产版本

```bash
npm run build
```

构建产物位于 `Frontend/dist/` 目录。

---

### 3. AI 服务 (gRPC)

#### 3.1 安装依赖

```bash
cd GlmAI
pip install -r requirements.txt
```

#### 3.2 配置 API Key

创建 `GlmAI/.env` 文件：

```env
# 智谱 AI API Key（必填）
GLM_API_KEY=your_zhipu_api_key_here

# GLM-4V API Key（可选，用于图片分析）
GLM_4V_API_KEY=your_glm4v_api_key_here

# 智谱知识库 ID（可选）
ZHIPU_KNOWLEDGE_ID=2041426353097789440
```

> **获取 API Key**: 访问 [智谱开放平台](https://open.bigmodel.cn/) 注册并创建 API Key

#### 3.3 启动服务

```bash
cd GlmAI/connect
python server.py
```

     5173 Listen
    50053 Listen
    50054 Listen

## 常见问题

### Q1: 后端启动报错 "ModuleNotFoundError"

**原因**: 依赖未安装或 Python 环境不正确

**解决方案**:
```bash
pip install -r requirements.txt
```

### Q2: 前端启动报错 "Port 5173 is in use"

**原因**: 端口被占用

**解决方案**:
```bash
# 方式一：Vite 会自动切换到下一个可用端口
# 方式二：手动终止占用进程
# Windows PowerShell
Get-NetTCPConnection -LocalPort 5173 | ForEach-Object { Stop-Process -Id $_.OwningProcess }
```

### Q3: AI 服务启动卡住不动

**原因**: 终端输出缓冲，服务实际上可能已启动

**解决方案**: 访问健康检查端点确认
```bash
curl http://localhost:50054
```
如果返回 `{"status": "healthy", ...}`，说明服务已正常启动。

### Q4: AI 服务报错 "未找到 GLM_API_KEY"

**原因**: 未配置智谱 API Key

**解决方案**: 创建 `GlmAI/.env` 文件并配置 `GLM_API_KEY`

### Q5: 数据库连接失败

**原因**: MySQL 服务未启动或配置错误

**解决方案**:
1. 确保 MySQL 服务正在运行
2. 检查 `Backend/.env` 中的数据库配置
3. 确保数据库 `carelink` 已创建

### Q6: 前端请求后端 404

**原因**: Vite 代理配置或后端路由问题

**解决方案**:
1. 确认后端服务已启动
2. 检查 `Frontend/vite.config.js` 中的代理配置
3. 前端请求路径应为 `/api/xxx`，代理会自动去掉 `/api` 前缀

## 生产环境部署建议

### 后端服务

```bash
# 使用 Gunicorn + Uvicorn
gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
```

### 前端服务

```bash
# 构建静态文件
npm run build

# 使用 Nginx 托管
# nginx.conf 示例:
# server {
#     listen 80;
#     server_name your-domain.com;
#     root /path/to/Frontend/dist;
#     location / {
#         try_files $uri $uri/ /index.html;
#     }
#     location /api {
#         proxy_pass http://localhost:8000;
#     }
# }
```

### AI 服务

```bash
# 使用 systemd 管理 (Linux)
# /etc/systemd/system/ai-service.service
# [Unit]
# Description=CareLinkAgent AI Service
# After=network.target
#
# [Service]
# Type=simple
# User=www-data
# WorkingDirectory=/path/to/GlmAI/connect
# ExecStart=/usr/bin/python server.py
# Restart=always
#
# [Install]
# WantedBy=multi-user.target
```
